What I‚Äôm Building

Under the hood, the watch is a layered embedded system:

  • ESP32-S3 platform driving a small color display with a custom UI built in LVGL
  • Touch interaction + gesture navigation (page switching, mode toggles, note cycling)

Right now, the watch is already at the point where it feels usable: you can swipe between pages, store multiple company notes, search and jump to a specific company, and keep accurate time across power cycles.

Technical Focus

This project is my sandbox for combining embedded systems, UX, and product thinking. Some of the areas I‚Äôm deliberately pushing on:

  • Embedded UI performance + responsiveness: keeping interactions smooth while layering multiple screens and animations
  • Firmware architecture: building the watch like a real product (state, persistence, commands, UI updates) instead of a monolithic loop
  • BLE protocol design: designing a simple command system that works in nRF Connect and scales to real apps
  • Persistence + reliability: saving notes/time safely, handling edge cases, and making sure the device recovers cleanly after resets
  • RTC integration: migrating from a purely software-ticked clock to a real RTC and debugging library/API mismatches
  • Human usability constraints: making sure the UI works for real use (long notes, scrolling behavior, quick access to the right info)

Challenges & What I Learned

A big part of this project has been dealing with real engineering friction

  • RTC integration wasn‚Äôt plug-and-play: library API mismatches, compile errors, and syncing logic issues forced a deeper understanding of I2C bring-up and time synchronization across power cycles.
  • Gesture UX tradeoffs: balancing scrolling within a long note while preserving swipe gestures for switching companies was a subtle but very real interaction design problem on a tiny screen.
  • Companion app evolution: moving from nRF Connect to a structured UI required treating BLE commands like a real product API, not just a debug interface.
  • AI note generation pipeline: early outputs were too vague to be useful, which pushed improvements in prompt design and output constraints to prioritize concrete conversation starters over generic company summaries.
  • Frame design: While I have had plenty of experience working with SOLIDWORKS and mechnical design, doing something at a small scale was a new challenge as it made me think about deformities, warping, and if it could print that you don't need to consider when working with metals like my other projects.
