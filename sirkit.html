Legend of Sir Kit

Turning a low-power TM4C123 into a tiny handheld game console with graphics, sound, and interrupts.

Overview

Legend of Sir Kit was my attempt to turn a low-power microcontroller into a full game console. Using a TM4C123, I wrote low-level C drivers to read a joystick via the ADC, handle buttons with GPIO interrupts, and send graphics over SPI to an ST7735 LCD. Periodic timers drove the game loop while edge-triggered interrupts kept movement responsive.

I reused my DAC work to generate in-game sound effects with a timer-driven audio engine and added quality-of-life features like multilingual text and score tracking. The fun part was squeezing everything into limited RAM and CPU cycles while still making the game feel smooth. It forced me to think in terms of registers, timing, and interrupt priority instead of just ‚Äúgame logic.‚Äù

How It Works

Under the hood, the ‚Äúconsole‚Äù is just a carefully wired set of peripherals and state machines:

  • The joystick uses two ADC channels; I map their values to player movement and dead zones.
  • Buttons generate edge-triggered GPIO interrupts to handle jumps, attacks, and menu actions.
  • A periodic timer interrupt ticks the game loop, updating positions, collisions, and scores.
  • The ST7735 LCD is driven over SPI with custom draw routines for sprites, HUD, and text.
  • A DAC + timer pair outputs simple waveforms for hits, pickups, and UI sounds.
  • All text strings are stored in small language tables so I can flip the UI between multiple languages.

Because everything runs on a small microcontroller, I spent a lot of time trimming sprite updates, reducing redraw regions, and making sure no interrupt ever blocked long enough to cause visible input lag.

Technical Highlights

  • Embedded C game engine running on TM4C123 with Keil ¬µVision.
  • Joystick input via ADC and digital buttons via GPIO interrupts.
  • Graphics rendered to ST7735 LCD over SPI with custom sprite and tile routines.
  • Timer-driven DAC audio for sound effects and simple music cues.
  • Finite state machines for game states, menus, and enemy behavior.
  • Multilingual UI and score tracking stored in lightweight data structures.
  • Careful CPU and memory budgeting to keep frame updates responsive on a constrained MCU.
