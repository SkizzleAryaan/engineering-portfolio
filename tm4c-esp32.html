Networked IoT Alarm Clock

A TM4C123 + ESP8266 alarm clock that syncs time, themes, and alarms with a browser UI over MQTT.

Overview

This project started as a course lab and turned into a full IoT system: a networked alarm clock built on a TM4C123 microcontroller and an ESP8266 Wi-Fi module. I wrote the C firmware that talks to the ESP over UART, parses MQTT over TCP, and keeps the MCU‚Äôs notion of time, alarm state, theme, and light level in sync with a browser-based UI.

On the hardware side, I integrated a SysTick-driven software clock, an ST7735 LCD driver, debounced push-button input, and a 4-bit R-2R DAC that plays different alarm melodies depending on the selected theme. I also wired in a 12-bit ADC and photosensor so the system can sense ambient light and reflect it on both the LCD and the web app.

The most satisfying part was seeing everything line up end-to-end: a time change on the web page publishes to the broker, the ESP forwards it, the TM4C updates its state, and the LCD and audio respond in real time. I even added two visual themes with different animations inspired by Deltarune, so the clock feels more like a tiny game console than a lab assignment.

How It Works

Under the hood, the system is a small distributed stack:

  • The web app (HTML/iFrame UI) publishes time, alarm, theme, and light commands to an MQTT broker.
  • The ESP8266 subscribes to those topics, forwards messages over UART, and relays MCU status updates back up to the broker.
  • The TM4C123 firmware parses those UART frames, updates internal state, and drives the LCD, DAC, and LEDs accordingly.
  • A SysTick-based software clock tracks time, while a 12-bit ADC channel monitors the photoresistor to infer ambient light level.

Locally, users can still control the clock using push-buttons: setting time, toggling alarms, or silencing the buzzer. Those actions propagate back through the ESP and MQTT so the browser UI never drifts out of sync.

Technical Highlights

  • Embedded C firmware on TM4C123 using Keil ¬µVision
  • UART protocol between TM4C123 and ESP8266 for MQTT/TCP message passing
  • 4-bit R-2R DAC and timer-driven sound engine for alarm melodies
  • 12-bit ADC + photoresistor for ambient light sensing and UI feedback
  • ST7735 LCD graphics routines for time, theme, and animation rendering
  • Web UI integrated via HTML/iFrames, synchronized through an MQTT broker
