TREL Thrust Vector Control System

From a 5000 lb gimbal test stand in SOLIDWORKS to real-time TVC firmware on an NI sbRIO.

Overview

When I joined the Texas Rocket Engineering Lab (TREL), I actually started on the mechanical side. My first job was helping design a 5000 lb-rated gimbal test stand in SOLIDWORKS to simulate hot-fire loads on our thrust vector control (TVC) system. Going through design reviews with NASA and Virgin-Galactic-affiliated mentors meant justifying every bolt, beam, and free-body diagram. It was my first taste of how serious real flight hardware design is.

From there I shifted into what I really love: firmware and control. I wrote embedded C/C++ control software for a 2-axis TVC system running on an NI sbRIO, starting from ‚Äúmake this actuator move‚Äù on an Arduino and ending with a full closed-loop control pipeline. I tuned the loop to run at 100 Hz with <10 ms command-to-actuation latency, wrote Bash scripts to flash and sync code across five flight computers in Linux, and built a valve-control state machine that sequenced pressurization, purges, and aborts across all launch phases. Working with GNC and Structures forced me to tie software decisions back to real valve dynamics, load paths, and failure modes.

How It Works

The TVC system is a 2-axis gimbal that steers the rocket‚Äôs engine by a few degrees in pitch and yaw to control attitude during flight. On the software side, I helped build a real-time control stack:

  • Sensor & command path: sbRIO reads actuator feedback and guidance commands, then computes new gimbal angles at 100 Hz.
  • Actuator control: PWM-driven servos convert those angles into physical motion, while limits and rate clamps keep the motion safe.
  • Valve state machine: a firmware FSM sequences pressurization, chill-down, purge, and abort states for the full hot-fire timeline.
  • Deployment: Bash scripts flash & synchronize the same image across five flight computers to keep the stack consistent before testing.

The result is a system where a change in guidance input propagates through real-time software and comes out as a smooth, controlled engine vector‚Äîon the test stand first, and eventually in flight.

Technical Highlights

  • Developed embedded C/C++ firmware for a 2-axis TVC system on an NI sbRIO-9637 controller.
  • Tuned a 100 Hz control loop with <10 ms end-to-end command-to-actuation latency.
  • Implemented PWM-based actuator control and safety limits for gimbal motion.
  • Built a valve-control finite state machine for pressurization, purge, and abort sequences.
  • Wrote Bash automation to flash and synchronize firmware across 5 flight computers in Linux.
  • Co-designed a 5000 lb-rated structural test stand in SOLIDWORKS, approved at CDR by NASA/Virgin Galactic reviewers.
  • Collaborated with GNC and Structures teams to align control software with real hardware loads and failure modes.
