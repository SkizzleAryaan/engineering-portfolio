Java Auction System

A multi-client auction platform with a Java GUI, socket-based networking, and MongoDB persistence.

Overview

In my software design course I built a full auction platform in Java with separate client and server programs. The server exposes a socket-based API that supports multiple concurrent bidders, while the client provides a desktop UI for browsing items, placing bids, and tracking live auctions.

On the backend, I implemented a multi-threaded architecture that handles each client connection in its own thread, using synchronized data structures to keep auctions, bids, and user sessions consistent. On the frontend, I built a Java GUI inspired by some of my favorite games so it feels more like a real app than a class demo.

System Design

  • Database: MongoDB to persist users, items, and bid history so auctions survive restarts.
  • Protocol: Text-based messages over TCP sockets to send commands and server responses.

What I Did & Learned

  • Implemented the multi-threaded Java server to handle multiple bidders at once.
  • Used synchronized data structures to keep auction state consistent across threads.
  • Built the Java client UI for accounts, listings, bidding, and live updates.
