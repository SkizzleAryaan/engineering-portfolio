Hardware Checkout Web App

A full-stack Flask + React system for managing lab equipment loans, roles, and overdue gear.

Overview

For a software engineering lab, my team and I built a hardware checkout web app to manage lab equipment loans. I worked across the stack: a Python Flask backend, a React frontend, and a MongoDB database for inventory, users, and transaction history.

The goal was to replace ad-hoc spreadsheets with a proper system where students can browse available hardware, submit checkout requests, and return items, while TAs and admins can approve, deny, and track everything in one place.

System Design

The app is built as a standard full-stack web system:

  • Backend: Flask REST API with endpoints for items, users, checkouts, and approvals, backed by MongoDB.
  • Frontend: React single-page app with views for browsing inventory, viewing your loans, and performing actions like request, approve, or return.
  • Auth & Roles: Role-based permissions so students, TAs, and admins see different actions and pages.
  • Deployment: Deployed to Heroku, with GitHub for version control and Jira for sprint planning and task tracking.

It felt like a mini product team: tickets, standups, code reviews, and a live web app that other people could actually break.

What I Did & Learned

  • Implemented Flask REST endpoints for check-in/out, user accounts, and role-based actions.
  • Helped design the MongoDB schema for inventory, users, and checkout history.
  • Built React components for inventory browsing, loan status, and admin approval flows.
  • Integrated the frontend with the backend via JSON APIs and Axios-style HTTP calls.
  • Deployed the app to Heroku and debugged issues across dev vs. production environments.
  • Used GitHub + Jira to collaborate in a structured, sprint-based workflow.

This project was the first time I got to ship something that felt like a real internal tool: log in, grab hardware, return it later, and have everything tracked without anyone touching Excel.
