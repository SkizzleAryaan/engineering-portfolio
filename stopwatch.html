FPGA Digital Stopwatch & Timer

A multi-mode stopwatch/timer built on a Basys3 FPGA using Verilog, finite state machines, and 7-segment LED control.

Overview

In this project I built a multi-mode digital stopwatch/timer on a Basys3 FPGA. I wrote the core logic in Verilog using finite state machines to handle four different operating modes (stopwatch, countdown, and more) while keeping the design synthesizable and readable.

I divided down the on-board system clock to get 10 ms timing resolution, drove the 7-segment LED displays with a multiplexing scheme, and mapped all of the switches and buttons using Xilinx constraint (.xdc) files. Using Vivado, I went from RTL to bitstream, ran functional and timing simulations, and learned how small changes in my state machine or reset logic could break the entire design.

It was my first real exposure to thinking in cycles and signals instead of lines of code, and it completely changed how I think about timing, resources, and hardware behavior.

How It Works

At a high level, the stopwatch is a collection of synchronous modules tied together by a top-level FSM:

  • A clock divider module scales the Basys3 100 MHz clock down to a 10 ms tick used for timekeeping.
  • A finite state machine controls the four operating modes, start/stop behavior, and when the counters increment or reset.
  • BCD counters track minutes and seconds and feed into the display driver.
  • A 7-segment multiplexing driver rapidly scans through digits so all four displays appear illuminated simultaneously.
  • All buttons and switches are mapped in a .xdc constraint file, tying logical signals to actual Basys3 I/O pins.

Most of the debugging happened at the waveform level in Vivado. Watching the FSM states, enables, and reset signals line up with the 10 ms tick taught me how sensitive real hardware is to off-by-one cycles and asynchronous resets.

Technical Highlights

  • Designed Verilog modules for clock division, FSM control, BCD counters, and 7-segment display drivers
  • Implemented four operating modes with a synthesizable finite state machine
  • Configured Basys3 I/O via Xilinx .xdc constraints for switches, buttons, and LEDs
  • Used Vivado for RTL simulation, timing analysis, and bitstream generation
  • Debugged reset logic, metastability issues, and off-by-one timing bugs at the waveform level
